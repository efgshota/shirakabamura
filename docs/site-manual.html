「CMS」と「静的」の違い：
CMS のページは MicroCMS から内容を更新できます。
静的 のページは開発者による修正が必要です。

URL変更履歴（旧URL → 新URL）

旧URLへのアクセスは自動的に新URLへ301リダイレクトされます。ブックマークや共有リンクは差し替えをお願いします。

ページ旧URL新URL変更理由
物件事例一覧 /business/ /stories/ 内容（開業ストーリー）に合わせた名称に変更
事例詳細 /business/[ID]/ /stories/[slug]/ URLパスの変更 + 可読性の高いslugに切り替え
お役立ち帳 /useful/ /info/ シンプルで直感的な名称に変更
物件詳細 /property/[自動ID]/ /property/[slug]/ ランダムIDから可読性の高いslugに切り替え

URL識別子の命名ルール

各コンテンツのURLは、slug フィールドまたはコンテンツIDによって決まります。

エンドポイントURL識別子形式例
お知らせ（/news/） コンテンツID 公開日（YYYYMMDD）＋ハイフン＋連番 20260301-1
物件事例（/stories/） slugフィールド 施設名の英語表記（小文字・ハイフン区切り） hygge、kariomon
物件（/property/） slugフィールド 地番または場所を示す英数字 kashiwabara-19-16
お知らせはコンテンツIDがURLになります：
ニュースには slug フィールドがないため、記事作成時に設定するコンテンツIDがそのままURLのパスになります。コンテンツIDは作成後に変更できないため、最初から YYYYMMDD-N 形式で入力してください。

物件・事例はslugフィールドが未設定の場合：
MicroCMS の自動生成ID（例: gu0jqlpsvn1）がそのままURLになります。記事登録時に slug フィールドを入力することを推奨します。

4. コンテンツの更新方法（MicroCMS）

MicroCMS とは

MicroCMS は日本製のコンテンツ管理システムです。ブラウザからログインして、物件情報やニュースなどを追加・編集・削除できます。変更は自動的にサイトに反映されます。

ログイン方法

  1. MicroCMS にアクセス
    https://app.microcms.io をブラウザで開く
  2. ログイン
    メールアドレスとパスワードでログイン
  3. サービスを選択
    白樺村のサービスをクリック
  4. APIを選択
    編集したいコンテンツ種別（物件 / 事業者 / お役立ち / ニュース）をクリック

10. 公開・デプロイの仕組み

サイト更新の2つの方法

A. コンテンツの更新（MicroCMS）

物件・事業者・お役立ち・ニュースなど、データの追加・編集・削除はMicroCMSから行います。コードの変更は不要です。

MicroCMSで編集 → 公開ボタン → サイトに自動反映

B. デザイン・機能の更新（コード変更）

レイアウト変更・新機能追加・デザイン修正など、サイトの見た目や動作を変える場合はコード修正が必要です。

コード修正 → PR作成 → レビュー → マージ → 自動デプロイ

メール送信用 DNS 設定（Resend）

お問い合わせフォームのメール送信（noreply@shirakabamura.com）を正常に届けるため、以下の DNS レコードも必要です。

Domain Verification（DKIM）
TypeNameContent
TXTresend._domainkeyp=MIGfMA...QIDAQAB
DKIM 署名用の公開鍵。メールの送信元を証明するために必要です。実際の値は Resend 管理画面で確認してください。
SPF（送信許可）
TypeNameContentPriority
MXsendfeedback-smtp.us-east-1.amazonses.com10
TXTsendv=spf1 include:amazonses.com ~all—
DMARC（任意・推奨）
TypeNameContent
TXT_dmarcv=DMARC1; p=none;

DNS 設定の確認手順

  1. DNS反映を待つ
    変更後、反映まで数分〜最大48時間かかります（通常は30分以内）
  2. SSL証明書
    Vercel が自動でSSL証明書（HTTPS）を発行します。手動設定は不要です
  3. 動作確認
    https://shirakabamura.com にアクセスして新サイトが表示されることを確認
  4. メール送信テスト
    お問い合わせフォームからテスト送信し、メールが正常に届くことを確認
切り替え時の注意：
DNS変更は全世界に反映されるまでタイムラグがあります。旧サイトと新サイトが混在する時間帯が発生するため、アクセスが少ない時間帯（深夜〜早朝）での切り替えを推奨します。
